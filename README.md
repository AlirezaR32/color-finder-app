# رنگ‌یاب — Color Finder

اپلیکیشن موبایلی (PWA) که با دوربین گوشی، رنگ هر سطحی را می‌خواند، کد دقیق آن (HEX / RGB) را استخراج می‌کند و نزدیک‌ترین نام واقعی رنگ را به **فارسی و انگلیسی** نشان می‌دهد.

## روش انتخاب‌شده و چرایی آن

هدف این بود که نام رنگ، نزدیک‌ترین حالت ممکن به برداشت چشم انسان از رنگ باشد، نه فقط نزدیک‌ترین عدد در فضای RGB. به همین دلیل:

1. **فضای رنگی CIE Lab (نه RGB)** — مقدار خام دوربین (RGB) ابتدا به فضای `Lab` با نور مرجع استاندارد D65 تبدیل می‌شود. این فضا برای تطبیق با درک بینایی انسان طراحی شده، برخلاف RGB که فقط برای نمایشگرها مناسب است.
2. **CIEDE2000 به‌جای فاصلهٔ اقلیدسی** — برای پیدا کردن نزدیک‌ترین رنگ در فهرست، به‌جای محاسبهٔ فاصلهٔ ساده، از فرمول **CIEDE2000** (Sharma, Wu & Dalal, 2005) استفاده شده؛ همان معیاری که در صنایع رنگ، نساجی و چاپ برای سنجش «تفاوت رنگی قابل‌مشاهده برای چشم انسان» استفاده می‌شود و دقیق‌تر از نسخه‌های قدیمی‌تر (CIE76/94) است. صحت پیاده‌سازی آن در این پروژه با جدول مرجع رسمی مقالهٔ Sharma و همکاران تست و تطبیق داده شده است.
3. **فهرست دوزبانهٔ رنگ‌ها** — ۵۰ رنگ پایه با نام اصیل فارسی و انگلیسی (مثل سرمه‌ای/Navy، فیروزه‌ای/Turquoise، نفتی/Petrol، کهربایی/Amber، زرد کره‌ای/Butter Yellow) انتخاب و هرکدام به ۵ حالت روشنایی/اشباع («بسیار روشن»، «روشن»، حالت پایه، «تیره»، «بسیار تیره») بسط داده شده‌اند تا کل طیف رنگی به‌صورت متراکم پوشش داده شود (در مجموع ۲۱۶ رنگ).
4. **میانگین‌گیری نویززدا** — به‌جای خوانش یک پیکسل، میانگین یک ناحیهٔ ۲۶×۲۶ پیکسلی در مرکز نشانگر گرفته می‌شود تا نویز سنسور دوربین کم‌تر اثر بگذارد.

> ⚠️ **محدودیت ذاتی روش دوربین:** نور محیط، تعادل سفیدی (white balance) و کیفیت سنسور دوربین گوشی روی رنگ خام اثر می‌گذارند. اپ همیشه میزان «دقت تطبیق» (مقدار ΔE) را هم نشان می‌دهد تا مشخص باشد رنگ یافت‌شده چقدر به رنگ واقعی نزدیک است. برای بهترین نتیجه، در نور روز یا نور سفید یکنواخت استفاده کنید.

## ساختار پروژه

```
color-finder-app/
├── index.html          صفحهٔ اصلی (ویوفایندر دوربین + پنل خوانش)
├── manifest.json        مانیفست PWA (برای نصب روی صفحهٔ اصلی گوشی)
├── sw.js                 Service Worker برای کش آفلاین
├── css/
│   └── styles.css        تمام استایل‌ها
├── js/
│   ├── colors.js          موتور رنگ: فهرست دوزبانه + تبدیل Lab + CIEDE2000
│   └── app.js             کنترل دوربین، نمونه‌گیری زنده و رابط کاربری
└── icons/
    ├── icon-192.png
    ├── icon-512.png
    └── icon-maskable-512.png
```

## اجرای محلی (Local run)

دوربین مرورگر فقط در «بستر امن» (HTTPS یا `localhost`) کار می‌کند؛ باز کردن مستقیم فایل با `file://` در بیشتر مرورگرها مسدود می‌شود. یکی از روش‌های زیر را اجرا کنید:

```bash
# با پایتون (از پیش روی اکثر سیستم‌ها نصب است)
cd color-finder-app
python3 -m http.server 8000
# سپس در گوشی/مرورگر باز کنید: http://localhost:8000
```

```bash
# یا با Node.js
npx serve color-finder-app
```

برای تست روی گوشی واقعی، گوشی و کامپیوتر باید در یک شبکهٔ Wi-Fi باشند و آدرس IP محلی کامپیوتر را به‌جای `localhost` وارد کنید (مثلاً `http://192.168.1.23:8000`). برخی مرورگرهای موبایل برای IP محلی هم اجازهٔ دوربین می‌دهند؛ اگر ندادند، مرحلهٔ بعد (دیپلوی روی HTTPS واقعی) را انجام دهید.

## نصب روی گوشی به‌صورت اپ واقعی (PWA)

برای دسترسی همیشگی به دوربین و نصب روی صفحهٔ اصلی، بهتر است پروژه را روی یک میزبان HTTPS رایگان قرار دهید:

1. **GitHub Pages** — مخزن را در گیت‌هاب بسازید، فایل‌ها را push کنید، از تنظیمات مخزن گزینهٔ Pages را فعال کنید.
2. یا **Netlify / Vercel** — پوشهٔ `color-finder-app` را در سایت آن‌ها drag-and-drop کنید (نیازی به دستور خاصی نیست).

بعد از باز شدن آدرس HTTPS در مرورگر موبایل:
- **اندروید (Chrome):** منو ⋮ → «Add to Home screen» / «نصب اپ»
- **آیفون (Safari):** دکمهٔ Share → «Add to Home Screen»

اپ به‌صورت تمام‌صفحه و بدون نوار آدرس مرورگر باز می‌شود، دقیقاً مثل یک اپ نصب‌شده.

## شخصی‌سازی

- برای افزودن رنگ‌های بیشتر، آیتم جدید به آرایهٔ `BASE_COLORS` در `js/colors.js` اضافه کنید (فقط نام فارسی، انگلیسی و کد HEX کافی است؛ تبدیل Lab و حالت‌های روشن/تیره به‌صورت خودکار ساخته می‌شوند).
- برای تغییر شدت تیره/روشن‌شدن خودکار، مقادیر `dS`/`dL` در آرایهٔ `VARIANTS` را تغییر دهید.
- رنگ‌های ظاهری برنامه (پس‌زمینه، رنگ تأکیدی و…) همگی در ابتدای `css/styles.css` به‌صورت متغیرهای CSS (`:root`) تعریف شده‌اند.

## فناوری به‌کار رفته

HTML / CSS / Vanilla JavaScript — بدون هیچ فریم‌ورک یا وابستگی بیرونی (به‌جز فونت‌های وب Vazirmatn و IBM Plex از Google Fonts). همین سادگی باعث می‌شود کل پروژه به‌سرعت لود شود و به‌صورت آفلاین هم (پس از اولین بازدید، به‌لطف Service Worker) کار کند.
